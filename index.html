Basic Usage

To add Split Inputs in your page all you need to do is add the following script tag at the bottom of the page & define a split input
<script src="split-inputs.js" onload="splitInputs.init({});"></script>
the object passed in splitInputs.init can contain options to determine global behaviour. default options are {disableCE:false,loadedStyles:false} , which disable customElements mode regardless of browser support and skip loading of default css styles ( in case you are going to fully override it or create your own css bundle).
To define a split input

define a split input with 6 boxes & allows for digits only (easy setup)
<split-input count="6" type="numeric"></split-input>

define a split input with 4 boxes & allows for alphabets only & has an extra attribute to access when styling in css
<split-input custom-style count="4" type="alpha"></split-input>

define a split input with custom html inside & allows for entering alphanumeric characters (powerful for styling)
<split-input type="alphanumeric">
<input type="text"><input type="text"><input type="text"><input type="text">
</split-input>

Basic Behaviour

  • Automatically moves focus to the next box when typing
  • Supports arrow keys to move between boxes for corrections
  • Backspace on empty box will focus previous box
  • Typing in a filled box will overwrite the character in it
  • Supports pasting an entire OTP in any box

Markup

  • Generate boxes using count attribute
  • Supports custom internal markup

Web Component

By default, split inputs tries to use a Custom Element & Shadow DOM. However, it will fallback to regular DOM elements if Custom Elements are unsupported.

Restyle Inputs

Default split input styles are loaded into the page automatically if the options object does not have loadedStyles:true, but you may want to overwrite some of these styles with your own custom styles, In that scenario you may access the individual Input elements using :host(selector) input for shadowdom mode and split-input inputfor DOM element mode just make sure to add a split-input attribute to the style element so that the node can be cloned for shadowdom mode. to style multiple split inputs on the same page, you can use a style tag similar to the following :
<style split-input>
:host([numeric-style]) input,split-input[numeric-style] input{
// your styles go here
}
</style>
this will apply your styles to any split-inputs with 'numeric-style' attribute added on them.

Events & JS API

Split-Inputs exposes 2 custom events splitInputs.CHANGE_EVENT and splitInputs.COMPLETE_EVENT fired when value in any of the input boxes changes & when all the boxes have a valid value respectively. in the raised events you can access either the combined values in the split input or each of the values individually.
// Event object
{detail:{
value:'value in split-input', // string
boxes:['value ','in ','split-input'] // array of individual box values
}}
// attach a listener
document.querySelector('split-input').addEventListener(splitInputs.CHANGE_EVENT,(v)=>console.log(v.detail));

Split-Inputs also has a simple api to set the value & focus on it as a whole, using element.splitInput.value & element.splitInput.focus(n = 0) when using element.splitInput.value you may assign to it or read from it, it will give you the concatenated value across all the individual boxes. element.splitInput.focus(n = 0) will focus on the nth box in the Split-Input , n has a default value of 0.
// update value and grab focus on 3rd element
document.querySelector('split-input').splitInput.value = "fill"
document.querySelector('split-input').splitInput.focus(3)
